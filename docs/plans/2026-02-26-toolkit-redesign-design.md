## 3. 图标方案

使用 lucide-react 库，每个工具对应一个 Lucide 图标。分类标题也用 Lucide 图标。

## 4. 统一设计系统（共享组件）

不强制布局模板，提供原子组件：

| 组件 | 用途 |
|------|------|
| `ToolPage` | 工具页面容器（标题栏 + 返回按钮 + 收藏 + 内容区） |
| `CodeBlock` | 带语法高亮的代码展示 + 复制按钮 |
| `TextInput` | 统一风格的输入 textarea |
| `TextOutput` | 统一风格的只读输出 textarea + 复制按钮 |
| `ActionBar` | 操作按钮栏 |
| `CopyButton` | 复制按钮（已有） |
| `FileDropZone` | 文件拖拽区域（桌面端工具用） |
| `ResultCard` | 结果展示卡片 |

## 5. 状态管理

### useToolkitStore（Zustand + localStorage）

```typescript
interface ToolkitState {
  favorites: string[];           // 收藏的工具 ID 列表
  recentUsed: string[];          // 最近使用（最多 10 个）

  toggleFavorite: (id: string) => void;
  addRecent: (id: string) => void;
```

## 6. 分类注册接口变更

```typescript
// 现有
type Category = "encoding" | "crypto" | "network" | "text" | "other";

// 新增
type Category =
  | "encoding"    // 编码转换
  | "crypto"      // 加密哈希
  | "text"        // 文本处理
  | "generator"   // 生成器
  | "datetime"    // 时间日期
  | "number"      // 数字进制
  | "network"     // 网络工具
  | "system";     // 系统工具

interface ToolPlugin {
  id: string;
  name: string;          // i18n key
  description: string;   // i18n key（新增：一句话描述）
  icon: ComponentType;   // Lucide React 组件（变更：从 string 改为组件）
  category: Category;
  component: ComponentType;
}
```

## 8. 技术要点

- 新增依赖：`lucide-react`
- 哈希 MD5/SM3、加密 AES/DES/SM4、Bcrypt：需要新增 Rust 后端命令
- 端口扫描、环境变量、系统信息：需要新增 Rust 后端命令
- 二维码：前端 JS 库（qrcode + jsQR）或 Rust 端
- 文本 Diff：前端 JS 库（diff）
- Cron 解析：前端 JS 库（cron-parser）
- 屏幕取色器、图片处理、文件哈希：复用已有 Rust 后端
